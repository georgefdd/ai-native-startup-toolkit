# 安全检查详表

## 使用说明
这是MVP上线前的安全检查详表。按照检查清单逐项审查，AI可以做第一轮检查，但涉及认证、密钥和数据处理的事项必须人工复查。

---

## 一、认证与会话管理

### 检查项
- [ ] 用户注册流程：密码强度要求、邮箱验证
- [ ] 登录流程：频率限制、暴力破解防护
- [ ] 密码存储：使用bcrypt/scrypt/argon2，不使用MD5/SHA
- [ ] Session管理：token过期、刷新机制、同时登录限制
- [ ] 权限控制：用户只能访问自己的数据
- [ ] 登出功能：正确清除session/token
- [ ] 密码重置：安全的重置流程，不泄露用户是否存在

### 常见AI生成代码的问题
- 硬编码的认证逻辑
- 不安全的token生成方式
- 缺少频率限制

---

## 二、数据暴露

### 检查项
- [ ] API响应不含不必要的字段（如密码hash、内部ID）
- [ ] 错误响应不泄露堆栈跟踪或内部路径
- [ ] 日志不记录敏感数据（密码、token、PII）
- [ ] CORS配置正确，不使用 `*`
- [ ] 文件上传有类型和大小限制
- [ ] 分页查询不暴露总记录数（除非必要）

### 常见AI生成代码的问题
- ORM默认返回所有字段
- 错误处理泄露内部信息
- 开发模式的详细错误在生产环境暴露

---

## 三、输入验证与注入

### 检查项
- [ ] 所有用户输入经过验证（类型、长度、格式）
- [ ] SQL查询使用参数化，不拼接字符串
- [ ] HTML输出经过转义（XSS防护）
- [ ] 文件路径验证（目录遍历防护）
- [ ] 命令注入防护（不直接执行用户输入）
- [ ] CSRF token验证
- [ ] Content-Type验证

### 常见AI生成代码的问题
- 直接拼接SQL
- 未转义的模板输出
- 信任客户端验证（服务端不重复验证）

---

## 四、依赖安全

### 检查项
- [ ] 运行依赖安全审计（npm audit / pip audit等）
- [ ] 无已知高危漏洞的依赖
- [ ] 依赖来自可信源
- [ ] 锁定依赖版本（package-lock / requirements.txt）
- [ ] 定期更新依赖
- [ ] 移除未使用的依赖

### 检查命令
```bash
# Node.js
npm audit
npm outdated

# Python
pip audit
pip check

# 通用
检查是否使用了已知有漏洞的依赖版本
```

---

## 五、密钥与配置管理

### 检查项
- [ ] API密钥不在代码仓库中
- [ ] 使用环境变量管理敏感配置
- [ ] .gitignore包含 .env、密钥文件等
- [ ] 生产密钥与开发密钥分离
- [ ] 密钥有轮换计划
- [ ] 第三方服务密钥权限最小化

### 绝对禁止
- ❌ 将密钥硬编码在代码中
- ❌ 将.env文件提交到git
- ❌ 在前端代码中暴露服务端密钥
- ❌ 使用默认密钥

---

## 六、基础设施

### 检查项
- [ ] 全站HTTPS
- [ ] 数据库不暴露在公网
- [ ] 有基本的速率限制
- [ ] 有备份策略
- [ ] 错误监控和告警
- [ ] 日志集中管理
- [ ] 容器/部署配置安全

---

## 安全审查优先级

| 优先级 | 范围 | 说明 |
|--------|------|------|
| P0 | 认证、密钥、数据处理 | 必须人工审查，上线前完成 |
| P1 | 输入验证、数据暴露 | AI+人工审查，上线前完成 |
| P2 | 依赖安全、基础设施 | AI审查为主，Launch前完成 |
| P3 | 日志、监控、备份 | 逐步完善，Scale前完成 |
