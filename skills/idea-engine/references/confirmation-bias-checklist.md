# 确认偏误自检清单

## 什么是确认偏误
确认偏误是创业者最大的认知陷阱：你倾向于寻找、解读和记住支持你已有信念的信息，同时忽略或贬低反面证据。

AI让确认偏误获得了"研究引擎"——问AI验证你的想法，它会找到支持证据；问它算市场规模，它会给出让TAM看起来很漂亮的数字。AI跟着你的方向走，而不是帮你找真相。

## 自检清单

### 在搜索信息时
- [ ] 你是否只搜索了支持你想法的关键词？
- [ ] 你是否跳过了看起来"负面"的搜索结果？
- [ ] 你是否只关注了成功的竞品，忽略了失败的？

**纠正**：主动搜索"[你的方向] 失败"、"[你的方向] 为什么不行"、"[类似产品] 倒闭"

### 在解读反馈时
- [ ] 你是否把模糊的正面反馈解读为强烈认可？
- [ ] 你是否把明确的负面反馈解读为"他不是目标用户"？
- [ ] 你是否更倾向于记住说好话的用户？

**纠正**：每条负面反馈的权重应该是正面反馈的3倍——因为人们出于礼貌更容易说好话。

### 在做市场分析时
- [ ] 你的TAM是否大得让任何投资人都兴奋？
- [ ] 你是否用了最乐观的增长假设？
- [ ] 你是否假设了你能拿到远超行业平均的市场份额？

**纠正**：用悲观假设重算一遍。如果悲观场景下生意仍然成立，那才靠谱。

### 在与AI对话时
- [ ] 你是否在用暗示性提问引导AI支持你？（"帮我看看为什么这个想法很好"）
- [ ] 你是否忽略AI提出的风险提示？
- [ ] 你是否反复修改提问直到AI给出你想要的答案？

**纠正**：始终先问"为什么这个想法可能失败"，再问"为什么可能成功"。

### 在做用户访谈时
- [ ] 你是否在访谈中介绍你的方案后才提问？
- [ ] 你是否在用户表达疑虑时下意识地为方案辩护？
- [ ] 你是否把"这听起来不错"等同于"我会付钱"？

**纠正**：访谈中80%时间用户说话，你只提问和追问。如果用户没提到方案解决的问题，那可能问题不够痛。

### 在做决策时
- [ ] 你是否因为已经投入了时间而拒绝pivot？（沉没成本）
- [ ] 你是否因为"没人做过"而兴奋而非警惕？
- [ ] 你是否在证据不足时用"直觉"填补空白？

**纠正**：把"直觉"替换为"假设"，然后验证它。

## 每周必答的3个问题

1. **本周收到的最强反面证据是什么？**
   - 如果想不起来，说明你在回避

2. **如果我的想法是错的，最可能错在哪里？**
   - 如果没有明确的"可能错在哪里"，说明你在自欺

3. **我是否在做只有我才看到的"好事"，还是市场也在告诉我同样的事？**
   - 如果只有你看到机会，要么你是先知，要么你错了

## 使用规则

- 每完成一个模块后，花5分钟过一遍这个清单
- 每周至少运行一次
- 如果连续两周没有任何反面证据出现，**这是最大的红色警报**
