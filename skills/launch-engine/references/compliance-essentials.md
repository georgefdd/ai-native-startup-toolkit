# 合规基础知识

## 常见合规框架速查

### GDPR（欧盟通用数据保护条例）
- **适用**：处理欧盟居民数据的任何公司
- **核心要求**：
  - 数据处理合法基础
  - 数据主体权利（访问、删除、携带）
  - 数据保护影响评估
  - 72小时数据泄露通知
  - 数据处理协议（与第三方）
- **MVP→Launch关键动作**：隐私政策、cookie同意、数据删除能力

### SOC 2（服务组织控制报告）
- **适用**：向企业客户销售SaaS产品
- **核心要求**：
  - 安全（Security）
  - 可用性（Availability）
  - 处理完整性（Processing Integrity）
  - 保密性（Confidentiality）
  - 隐私（Privacy）
- **MVP→Launch关键动作**：审计日志、访问控制、加密、事件响应计划

### HIPAA（美国健康保险流通与责任法案）
- **适用**：处理美国医疗健康信息的公司
- **核心要求**：
  - 保护PHI（受保护健康信息）
  - 技术保障、物理保障、行政保障
  - 业务伙伴协议（BAA）
- **MVP→Launch关键动作**：数据加密、访问控制、审计日志、BAA

### 等保（中国网络安全等级保护）
- **适用**：在中国运营的信息系统
- **核心要求**：
  - 按等级（1-5）实施安全保护
  - 网络安全、数据安全、应用安全
  - 定期测评
- **MVP→Launch关键动作**：等保定级、安全建设、测评备案

### 个保法（中国个人信息保护法）
- **适用**：处理中国境内自然人个人信息的任何组织
- **核心要求**：
  - 处理个人信息需获得同意
  - 最小必要原则
  - 个人信息跨境传输限制
  - 个人信息保护影响评估
- **MVP→Launch关键动作**：隐私政策、同意机制、数据分类分级

## 合规建设时间线

### Launch前必须完成
- [ ] 隐私政策
- [ ] 服务条款
- [ ] 数据加密（传输+存储）
- [ ] 基本访问控制
- [ ] 审计日志

### 企业客户签约前需要
- [ ] SOC 2准备（或等保）
- [ ] 安全白皮书
- [ ] 数据处理协议
- [ ] SLA模板
- [ ] 事件响应计划

### Scale阶段需要
- [ ] 正式合规认证
- [ ] 第三方安全审计
- [ ] 合规持续性监控

## 合规不是成本，是壁垒

在AI平权的世界里，合规能力本身是护城河。能通过企业采购审核的AI产品，比不能通过的多了整个企业市场。把合规从"成本中心"重构为"竞争优势"。
