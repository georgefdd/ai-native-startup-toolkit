# 运营自动化常见模式

## 模式1：反馈收集与分诊

**触发**：新用户反馈到达（邮件/表单/应用内）
**自动化流程**：
1. 接收原始反馈
2. 分类：bug报告 / 功能请求 / 使用问题 / 赞扬
3. 打标签：严重度、功能模块、用户类型
4. 提取关键信息：一句话摘要、情绪、紧急程度
5. 路由到正确的处理队列
6. 发送确认给用户

**人工检查点**：每日浏览分诊结果，审批P0项

## 模式2：周报自动编译

**触发**：每周一早上
**自动化流程**：
1. 从各数据源拉取指标（用户量、活跃度、收入、留存）
2. 计算周环比变化
3. 生成结构化摘要
4. 标注异常值和趋势变化
5. 放入共享文件夹/发送邮件

**人工检查点**：创始人审核后发布

## 模式3：客户Onboarding

**触发**：新用户注册
**自动化流程**：
1. 发送欢迎邮件（个性化：基于用户画像）
2. 触发引导教程序列
3. 监控激活进度
4. 未激活用户3天后发提醒
5. 7天未激活标记为"需要关注"

**人工检查点**：每周review未激活用户列表

## 模式4：内容发布日历

**触发**：定时/内容就绪
**自动化流程**：
1. 从内容库选取待发布内容
2. 按平台格式化
3. 排期发布
4. 发布后追踪表现
5. 生成内容效果周报

**人工检查点**：创始人审核每条内容（至少初期）

## 模式5：CRM同步

**触发**：交易状态变化
**自动化流程**：
1. 检测到交易阶段变化
2. 更新CRM记录
3. 触发下一步动作（发送跟进邮件/安排会议/更新管道）
4. 记录时间线

**人工检查点**：每周pipeline review

## 自动化实施原则

1. **先文档化流程，再自动化**——如果流程不在纸上，自动化只是在自动化混乱
2. **从半自动开始**——AI建议，人确认。稳定后再考虑全自动
3. **每个自动化有降级方案**——如果自动化失败，手动流程是什么？
4. **异常比正常更重要**——80%的自动化处理20%的正常情况，80%的精力应该放在20%的异常处理
5. **渐进委托**——先让AI起草，人审批；再让AI执行，人review；最后AI全自主，人抽查
